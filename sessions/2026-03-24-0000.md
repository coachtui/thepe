## Decisions

- Action bar only renders when message is complete and not loading (avoids flash during streaming)
- `correctionForm` is reset each time the modal opens to avoid stale values
- "Recheck Plans" button is disabled with `cursor-not-allowed` — Phase 7D will activate it once `recheck-workflow.ts` is built
- `X-Query-Id` is captured from response headers and stored with each assistant message; passed as part of correction payload (via `ai_response_excerpt` currently, not a separate field in the corrections schema)
- `✓ Correct` is local-state only — no write to DB; spec does not define what DB action this triggers
