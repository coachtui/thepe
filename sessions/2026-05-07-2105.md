Turn the 16 manual-review artifact cleanup cases into a product workflow with script-side marking and a UI review queue. Human-in-the-loop QA, not blind auto-cleaning.

```ts
artifactReviewStatus?: 'artifact_suspected' | 'resolved' | 'ignored'
artifactReviewReason?: string

```
These fields live in `item_payload` JSONB and are preserved through the read path via the `...base` spread in `mergeRowOntoItemPayload`. No schema migration needed.

### `scripts/clean-submittal-artifacts.ts` — MODIFIED
Added section 5c to the `--execute` path: marks manual_review rows in the DB.
- For each manual_review row: updates `item_payload` with `artifactReviewStatus: 'artifact_suspected'`, `artifactReviewReason`, and `artifactSuggestedName`
- Skips rows already marked `resolved` or `ignored` (idempotent re-runs)
- `submittal_item` column is NOT changed — the artifact text remains until a human accepts/edits
- Dry-run output unchanged (shows manual_review rows as before)
- Summary now shows "Queued for review" count alongside Deleted/Cleaned

### `src/app/api/projects/[id]/submittal-register/artifact-review/route.ts` — NEW
`POST` route for artifact review actions.
- **Auth**: any project member (member check identical to existing `/review` route)
- **Actions**: `accept` | `edit` | `ignore`
  - `accept`: uses `item_payload.artifactSuggestedName` → updates `submittal_item` + `item_payload.submittalItem`; sets `artifactReviewStatus: 'resolved'`
  - `edit`: uses caller-supplied `clean_name` → same updates; `resolved`
  - `ignore`: no name change; sets `artifactReviewStatus: 'ignored'`
- Returns `{ success, itemId, action, newSubmittalItem, artifactReviewStatus }`

### `src/components/submittal/ArtifactReviewQueue.tsx` — NEW
Collapsible review queue panel component.
- Filters `items` for `artifactReviewStatus === 'artifact_suspected'` (with local override to hide resolved/ignored this session)
- Shows count badge "N needs review" when collapsed
- Per-item display: section + page · current text (red mono) · suggested name (green mono) · review note · source text excerpt (truncated at 160 chars)
- Per-item actions: **Accept suggestion** (only shown when suggested name exists) · **Edit name** (inline text input) · **Ignore**
- Optimistic local state update on success; error display on failure
- Returns `null` when no artifact-suspected items exist (zero render cost for clean registers)

## How the Workflow Runs

1. Run `npx tsx scripts/clean-submittal-artifacts.ts --execute`
   - 4 rows deleted, 2 rows cleaned, **16 rows marked `artifact_suspected`** in `item_payload`
2. User opens the submittal register in the UI
3. An amber "Extraction Review Queue — 16 needs review" panel appears above the sections
4. User clicks to expand; sees each row with original artifact text, suggested clean name, review note, and source excerpt
5. For each row: Accept suggestion, Edit name, or Ignore
6. On action: `POST /artifact-review` updates DB, panel updates optimistically
7. When all 16 are resolved, the queue panel disappears
