## Files Created

### `scripts/clean-submittal-artifacts.ts`
One-time artifact cleanup script following the same pattern as `backfill-submittal-section-titles.ts`.

- **Dry-run default** — no writes unless `--execute` is passed
- **`--project=<uuid>`** arg support (defaults to the Ammunition project)
- **Latest-run-only scope** — queries workflow_runs for most recent completed run, then fetches all items via paginated `.range()` calls

**Detection logic:**
- Primary signal: item ends with trailing dashes after a break fragment — `BREAK---`, `REAK---`, `GE-BREAK---`, `E-BREAK---`, `AGE-BREAK---`, `--PAGE-BREAK---`
- The trailing-dashes requirement correctly excludes real words like "Breakers" and "Breaking" (no trailing dashes on those)
- Secondary: `--PAGE-BREAK---` embedded inline mid-string
- `rawExcerpt` is never touched

**Classification tiers:**
- `safe_delete` — stripped text is empty (entire item was artifact)
- `safe_clean` — stripped text has ≥ 3 words, last word is not a conjunction/preposition
- `manual_review` — stripped text is 1–2 words, ends with conjunction, or ambiguous

**Write actions (`--execute`):**
- `safe_delete`: DELETE the row from `submittal_register_items`
- `safe_clean`: UPDATE `submittal_item` + `item_payload.submittalItem` with stripped text; append cleanup notice to `item_payload.notes`
- `manual_review`: NO action — user decides
