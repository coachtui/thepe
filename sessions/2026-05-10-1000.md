## Goal
Implement source selection utility that evaluates narrative vs DD-form extraction and
chooses the better source (or hybrid of both) for UFGS specs.

## Files Created or Modified

### New: `src/lib/ingestion/submittal-source-selector.ts`
- `mapDDFormRowToSubmittalItem(row)` — converts DDFormRow → SubmittalRegisterItem shape.
  Sets `confidence: 0.92` (authoritative source), `approvalRequired: true` for G authority,
  `sourceReference.extractionSource: 'ufgs_dd_form'`, `blockingRisk: 'none'`.
- `chooseSubmittalExtractionSource(input)` — evaluates sources with 5-case decision tree:
  1. No DD-form rows → narrative
  2. DD-form SD% < 80% threshold → narrative
  3. Narrative SD% >= DD-form SD% → narrative
  4. DD-form covers < 70% of narrative spec sections → hybrid
  5. Otherwise → dd_form

### Source selection comparison

| Source | Items | SD Coverage | Auth Coverage | Selected |
|---|---|---|---|---|
| Narrative | 3,537 | 18.9% | 6.4% | |

**Why hybrid, not dd_form?**
DD-form covers 47 of 77 narrative spec sections (61%), which is below the 70% hybrid threshold.
The remaining 30 spec sections only appear in the narrative body text and need to be filled in.

**Improvements over narrative-only:**
- SD coverage: 18.9% → 36.3% (+17.4pp)
- Authority coverage: 6.4% → 22.9% (+16.5pp)
- Item count: 3537 → 1593 (more focused — narrative had many low-quality rows)

## Adversarial Fixtures
No change — TXT files use default mode with no source selection logic applied.

## Decisions
- Hybrid threshold 70%: if DD-form covers ≥ 70% of narrative spec sections, use pure DD-form.
  At 61% coverage for this UFGS spec, hybrid fills the 30 uncovered sections.
- Deduplication in hybrid: uses `simpleDedupeKey(spec, sdCode, submittalItem[:40])` to prevent
  the same (spec, SD code, item name) appearing from both DD-form and narrative.
- Evaluation mode maintained: no DB writes, no production extraction change.

## Unresolved Items
- The 1593 hybrid items include all narrative items for the 30 uncovered spec sections,
  which includes many low-quality narrative rows. A future improvement: apply the SD code
  threshold (e.g., only fill from narrative for sections where narrative has ≥ 50% SD coverage).
- The `14 00 01` false-positive spec section from CSI boundary concatenation means ~2 DD-form
  rows are attributed to the wrong spec section. These rows appear in the hybrid's DD-form portion.
