## Changes in Detail

### 1. vision-processor.ts — Manual Analyze button bug fix
`processDocumentWithVision()` (the function called by the Manual Analyze button) was missing the `indexDocumentPage()` call that the Inngest path (`processDocumentPageRange()`) already had. Without it, `document_pages` and `sheet_entities` were never populated for manually-analyzed documents, breaking plan reader sheet narrowing and (upcoming) recheck workflow.

Fix:
- Changed first chunk fetch from `select('id')` to `select('id, content')` to capture `chunkContent` for the indexer
- Added `indexDocumentPage()` call (wrapped in try/catch as non-fatal) before the callout step, capturing `documentPageId`
- Replaced `null` in `storeComponentCallouts()` call with the captured `documentPageId`

### 2. types.ts — EvidenceItem fields
Added two optional fields to `EvidenceItem`:
- `query_id?: string` — unique per chat request, threaded through all retrieval steps
- `source_confidence_at_retrieval?: number` — raw confidence before any modifier is applied

Added new interfaces:
- `DataSourceCounts` — `{ vision_db, vector, plan_reader, graph }`
- `ChatResponse` — top-level chat response metadata shape

### 3. chat-handler.ts — queryId, trace store, data_source_counts
- Added module-level `traceStore` (Map keyed by queryId, 30-min TTL) and `getStoredTrace()` export
- `handleChatRequest()` now generates a `queryId` via `randomUUID()` at request start
- After augmented packet is assembled, stamps `query_id` and `source_confidence_at_retrieval` onto all evidence items
- Calls `storeTrace()` to persist items in the in-memory store
- Computes `DataSourceCounts` by iterating evidence items and adjusting for plan reader items
- Attaches `X-Query-Id` and `X-Data-Source-Counts` headers to every response (trace + normal paths)

### 4. query-trace route
New `GET /api/projects/[id]/query-trace/[queryId]` endpoint:
- Auth: project membership required
- Returns `{ queryId, projectId, itemCount, items }` from trace store
- Returns 404 if trace not found or expired

## Decisions Made
- Trace store is in-process memory only — will not persist across serverless instances. Acceptable for dev/debug use. Persisting to DB would require a new table (Phase 7B scope).
- `source_confidence_at_retrieval` is set to `item.confidence` if not already set at retrieval time — ensures the field is always populated on stamped items even before retrieval steps explicitly set it.
- `DataSourceCounts.graph` is left at 0 for now — no graph retrieval source type exists in `EvidenceSourceType` yet. Will be populated when graph queries are wired through.
