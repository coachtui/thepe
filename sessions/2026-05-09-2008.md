# Session 2026-05-09-2008

## Goal
Implement first-pass Publish Register gate: publish readiness utility, UI banner with confirmation modal, and harness tests.

## Files Created
- `src/lib/chat/submittal-publish-readiness.ts` — pure `evaluateRegisterPublishReadiness` function. Accepts optional `ingestionGrade`, `ingestionGradeReasons`, and `qaResult`. Returns `{status, reasons, requiredActions}`. Rules: `poor_extraction` → blocked; critical QA → blocked; `needs_review` grade or warning QA → needs_review; otherwise ready.
- `src/components/submittal/PublishReadinessBanner.tsx` — React banner component. Renders colored status panel (green/amber/red) with reason list and Publish Register button. Blocked: button disabled. needs_review: opens confirmation modal with issue list before publishing. Manages local `publishedAt` state; shows "Unpublish" option after publish.

## Files Modified
- `src/components/submittal/tabs/OverviewTab.tsx` — Added `ingestionGrade?` and `ingestionGradeReasons?` props. Computes `qaResult` and `readiness` via `useMemo`. Renders `<PublishReadinessBanner>` at top of tab.
- `scripts/qa-submittal-harness.mjs` — Added import for `evaluateRegisterPublishReadiness`. Added 7 publish readiness tests (PRG-1 through PRG-7).

## Decisions
- `ingestionGrade` is not wired through SubmittalsCommandCenter → OverviewTab in this session. The harness grade is not stored per production run, so the UI operates readiness from QA findings only. The prop is ready to receive a grade if one is stored in the future.
- Publish state is local client state only (`publishedAt` in `PublishReadinessBanner`). No DB migration.
- No approvals routing, email, PDF generation, or server workflow added.
