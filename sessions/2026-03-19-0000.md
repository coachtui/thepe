## Files Created

### `src/app/api/projects/[id]/corrections/route.ts`
POST endpoint to capture user corrections to AI answers.
- Auth: project_members.role IN ('owner', 'editor')
- Writes to project_corrections (validation_status='pending' or 'accepted')
- Auto-accept logic: submitted_by_role weight >= 2.0 (PE=3.0, superintendent=2.0, admin=2.5)
  → also writes accepted project_memory_items row and links correction.memory_item_id

### `src/app/api/projects/[id]/memory/confirm/route.ts`
POST endpoint for confirm/dispute votes on project_memory_items.
- Auth: any project member
- Writes to memory_confirmations (UNIQUE prevents double-vote)
- Updates confirmed_by_count / rejected_by_count on parent memory item
- Sets validation_status='disputed' if rejected_by_count > confirmed_by_count
- Restores 'accepted' if votes swing back to confirmed majority
